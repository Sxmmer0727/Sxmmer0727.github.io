第一篇博客：基于Hexo和Github的博客搭建

2022-02-21 729 words
学习

成为程序员多年以来的第一篇博客，目前是浙江工业大学研二的一名学生，虽然在老师底下进行的研究是关于综合能源管理方面，更细一点就是光伏发电功率的预测，这样的研究取得了一点小成就，但基于对未来就业的考虑，还是决定吃JAVA这颗卷心菜！由于学习JAVA知识体系结构比较发杂，虽然有着比较好的编程基础，也系统的学习过编程。但是为了就业还是打算更加系统的学习并记录，所以产生了写博客的想法。

此前本人对博客一无所知，因此咨询了一下身边的子涵大佬，于是得到了这样的回答。



emmm我的博客目的只是作为一个学习的总结以及监督效果，目前花太大精力我还是不太愿意的，所以就从b站等地方下手寻找资源，最终采用了这个基于Hexo和Github的博客。

博客的搭建主要分为以下四步

博客搭建的主要步骤

  • 软件安装
  • Github的准备工作
  • 本地搭建博客
  • 博客上传以及更新维护

1.安装Node.js以及Git和Hexo

这些软件都有详细的安装方法，在这里就不做赘述。需要注意的是：Node.js的版本一定不能过低，否则在用指令安装Hexo的时候就会产生如下报错。


2.Github的准备工作

Github的准备工作主要就是新建一个responsitory作为博客的仓库，比较重要的一步就是responsitory name需要命名为 你的Github昵称.github.io

随后生成ssh公共密钥：在博客文件夹下右键点击Git Bush Here, 出现Git的控制台

$ ssh
$ ssh-keygen -t rsa -C"Github的邮件地址"

这之后可在电脑该用户名下找到.ssh文件夹，打开id_rsa.pub文件全选复制粘贴到Github的new SSH key 中即可,随后测试是否绑定成功

$ ssh -T git@github.com

3.本地搭建博客

在博客的文件夹下打开Git控制台，输入以下指令

$ hexo init // 初始化
$ hexo s // 打开本地服务器

在这里可以进行博客的model的美化，我从官网上找到了现在的这个主题，感觉还8错。需要修改配置文件中的主题名名称。

4.博客上传至服务器

修改配置文件中的type 、 repository 以及 branch属性

type: git
repository: 你的Github昵称.github.io
branch: main

调用Git控制台

$ npm install hexo-deployer-git --save // 安装部署工具
$ hexo g  // 生成
$ hexo d  // 部署

再通过Github的授权，由此就完成了博客的上传搭建，就可以用网络访问啦！
